What are Embeddings? START HERE
The core concept you must understand before writing a single line of embedding code.
One-line definition: An embedding model converts a piece of text into a list of numbers (a vector) — such that texts with similar meaning produce similar numbers. That's it.
1
Text goes in

"The weather in Bhubaneswar is hot and humid in summer"

2
Embedding model processes it

A neural network (transformer) encodes the semantic meaning of the sentence.

3
A vector comes out

[0.021, -0.134, 0.876, 0.003, ...] — typically 384 to 3072 numbers long.

4
Similar text → similar vectors

"Bhubaneswar gets very warm and moist in June" → nearly the same numbers. That's semantic similarity.

5
Stored in a vector database

These vectors for all your chunks are stored. At query time, the query is also embedded and the closest vectors are found — that's retrieval.

Visual — similarity in vector space

Each bar = one dimension of the embedding vector. Visually similar bar patterns = semantically similar text.

concept_demo.py — see embeddings in action
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")

texts = [
    "What is retrieval augmented generation?",   # query
    "RAG combines retrieval with language generation", # relevant
    "The sky is blue on a clear day",             # unrelated
]

embeddings = model.encode(texts)

print(f"Vector shape: {embeddings.shape}")
# Output: (3, 384)  ← 3 texts, each = 384-dim vector

# Cosine similarity between query and each text
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

query_emb = embeddings[0]
for i, text in enumerate(texts[1:], 1):
    sim = cosine_similarity(query_emb, embeddings[i])
    print(f"Similarity to '{text[:40]}...': {sim:.4f}")

# Output:
# Similarity to 'RAG combines retrieval with...': 0.8741  ← high = relevant
# Similarity to 'The sky is blue on a clear...':  0.1023  ← low = unrelated

# ── Key insight ───────────────────────────────────────
# In your RAG pipeline:
# 1. Embed all chunks → store vectors in vector DB
# 2. Embed user query → compare to stored vectors
# 3. Top-k closest vectors → these are your context chunks
# 4. Pass chunks + query to LLM → final answer
Why this matters for RAG: The quality of your embedding model directly determines retrieval quality. A bad embedding model finds the wrong chunks. A great one finds the exact right context. Everything else downstream (chunking, vector DB, LLM) depends on this step being correct.

How to Choose the Right Model
Use this decision tree to pick the right model for your situation — don't overthink it, the decision tree covers 95% of cases.
❓ What is your primary constraint?
No money / offline
→ BAAI/bge-small-en-v1.5 (HuggingFace, free, 133MB). Or nomic-embed-text via Ollama for long docs.
Learning / experiments
→ all-MiniLM-L6-v2 — fastest to set up, runs on any machine, great for understanding concepts.
Best free quality
→ BAAI/bge-large-en-v1.5 — best MTEB among free models, needs 4GB RAM.
Cheapest paid API
→ Google text-embedding-005 at $0.006/MTok — or voyage-4-lite at $0.02/MTok with 200M free tokens.
Best retrieval quality
→ voyage-4-large — best MTEB retrieval scores in 2026, 32K token context.
Multilingual content
→ Cohere embed-v4.0 — 100+ languages, or BAAI/bge-m3 (free, multilingual).
Code / technical docs
→ voyage-code-3 (paid) or Codestral Embed by Mistral for code-specific retrieval.
Very long documents
→ Cohere embed-v4.0 (128K tokens) or voyage-4-lite (32K). Both handle full docs without chunking.
OpenAI ecosystem already
→ text-embedding-3-small — simplest integration, battle-tested, good enough for most cases.
GDPR / European compliance
→ mistral-embed — EU-based, GDPR compliant by default.
Master tip: Start with bge-small-en-v1.5 (free, local) to build and test your pipeline. Only switch to a paid API when you've validated the pipeline works end-to-end. Switching embedding models requires re-embedding your entire document store — so don't switch casually once in production.

embed_e5.py — Microsoft E5 with query/passage prefixes
from sentence_transformers import SentenceTransformer
from langchain.schema import Document
import numpy as np

# ── E5 requires explicit prefixes for best performance ─
# Documents: "passage: your text here"
# Queries:   "query: your question here"

model = SentenceTransformer("intfloat/e5-base-v2")

def embed_documents_e5(chunks: list[Document]) -> np.ndarray:
    texts = [f"passage: {c.page_content}" for c in chunks]
    return model.encode(texts, normalize_embeddings=True)

def embed_query_e5(query: str) -> np.ndarray:
    return model.encode(f"query: {query}", normalize_embeddings=True)

# Usage
doc_vecs = embed_documents_e5(stage2_chunks)
query_vec = embed_query_e5("What is the return policy?")

# ── Compare query against all docs ────────────────────
similarities = np.dot(doc_vecs, query_vec)
top_k_indices = np.argsort(similarities)[::-1][:5]

print("Top 5 relevant chunks:")
for i in top_k_indices:
    print(f"  [{similarities[i]:.3f}] {stage2_chunks[i].page_content[:100]}")

import json, os, hashlib, time
from pathlib import Path
import numpy as np
from langchain.schema import Document

# ── Cache embeddings to avoid re-embedding ────────────
class EmbeddingCache:
    def __init__(self, cache_dir: str = "./embedding_cache"):
        self.cache_dir = Path(cache_dir)
        self.cache_dir.mkdir(exist_ok=True)

    def _hash(self, text: str) -> str:
        return hashlib.md5(text.encode()).hexdigest()

    def get(self, text: str):
        path = self.cache_dir / f"{self._hash(text)}.npy"
        if path.exists():
            return np.load(str(path))
        return None

    def set(self, text: str, embedding: np.ndarray):
        path = self.cache_dir / f"{self._hash(text)}.npy"
        np.save(str(path), embedding)

# ── Batch embedding with progress bar + caching ───────
def embed_chunks_batch(
    chunks: list[Document],
    embeddings_model,
    batch_size: int = 32,
    use_cache: bool = True,
    cache_dir: str = "./embedding_cache"
) -> list:
    """Embed chunks in batches with caching and progress tracking."""

    cache = EmbeddingCache(cache_dir) if use_cache else None
    all_embeddings = []
    cache_hits = 0
    to_embed = []

    # Check cache first
    for chunk in chunks:
        if cache:
            cached = cache.get(chunk.page_content)
            if cached is not None:
                all_embeddings.append(cached)
                cache_hits += 1
                continue
        to_embed.append(chunk)
        all_embeddings.append(None)   # placeholder

    print(f"Cache hits: {cache_hits}/{len(chunks)} | To embed: {len(to_embed)}")

    # Embed remaining in batches
    embed_idx = 0
    for i in tqdm(range(0, len(to_embed), batch_size), desc="Embedding"):
        batch = to_embed[i:i+batch_size]
        batch_texts = [c.page_content for c in batch]
        batch_vecs = embeddings_model.embed_documents(batch_texts)

        for j, vec in enumerate(batch_vecs):
            if cache: cache.set(batch[j].page_content, np.array(vec))

        # Fill in placeholders
        none_idx = [k for k, e in enumerate(all_embeddings) if e is None]
        for k, vec in zip(none_idx[:batch_size], batch_vecs):
            all_embeddings[none_idx[k]] = np.array(vec)

    print(f"✅ Embedded {len(all_embeddings)} chunks total")
    return all_embeddings

def estimate_embedding_cost(chunks: list[Document], model: str = "openai-small"):
    total_chars = sum(len(c.page_content) for c in chunks)
    approx_tokens = total_chars // 4   # rough estimate: 4 chars ≈ 1 token
    prices = {
        "openai-small": 0.02,   # per MTok
        "openai-large": 0.13,
        "voyage-lite":  0.02,
        "voyage-large": 0.06,
        "cohere":       0.12,
        "google":       0.006,
        "jina":         0.02,
    }
    price_per_mtok = prices.get(model, 0.02)
    cost = (approx_tokens / 1_000_000) * price_per_mtok
    print(f"Chunks: {len(chunks)}")
    print(f"Approx tokens: {approx_tokens:,}")
    print(f"Estimated cost ({model}): ${cost:.4f}")
    return cost
