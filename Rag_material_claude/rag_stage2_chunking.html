Quick concepts to understand first

concepts.py — read this before everything else
# ─────────────────────────────────────────────────────────
# CONCEPT 1: chunk_size
# How many characters (or tokens) per chunk.
# Too small → loses context. Too large → noisy retrieval.
# ─────────────────────────────────────────────────────────
chunk_size = 500   # ~80–120 words. Good starting point.
chunk_size = 1000  # ~200 words. Better for complex documents.

# ─────────────────────────────────────────────────────────
# CONCEPT 2: chunk_overlap
# How many characters are SHARED between consecutive chunks.
# Prevents losing context at chunk boundaries.
# Rule of thumb: 10–20% of chunk_size
# ─────────────────────────────────────────────────────────
chunk_overlap = 100  # for chunk_size=500
chunk_overlap = 200  # for chunk_size=1000

# ─────────────────────────────────────────────────────────
# CONCEPT 3: The output is ALWAYS list[Document]
# Every chunker returns the same type — a list of Documents.
# Each chunk becomes one Document with its own metadata.
# ─────────────────────────────────────────────────────────
# chunks[0].page_content  → the text of chunk 0
# chunks[0].metadata      → source, page, start_index, etc.

# ─────────────────────────────────────────────────────────
# CONCEPT 4: split_documents() vs create_documents()
# split_documents() → takes list[Document] (from Stage 1)
# create_documents() → takes list[str] (raw strings)
# Always use split_documents() in your pipeline.
# ─────────────────────────────────────────────────────────
Key Rule: chunk_overlap should never be ≥ chunk_size. Keep it at 15–20% of chunk_size. E.g., chunk_size=1000 → chunk_overlap=150 or 200.

Decision Tree — which chunker for which document?

❓ What type of document are you chunking?
PDF / text file
→ Use Recursive Character (Strategy 2). It's the best default.
Scanned PDF (OCR)
→ Use Sentence-Window (Strategy 8) — OCR text is noisy; sentence splitting handles it better.
Markdown docs / wikis
→ Use Markdown Header (Strategy 6). It preserves the section hierarchy.
Python / JS / HTML code
→ Use Code-Aware (Strategy 7). Never splits mid-function or mid-tag.
CSV / Excel rows
→ No chunking needed — rows are already chunks from Stage 1.
Research papers / long docs
→ Use Semantic Chunking (Strategy 4). Splits where topics change, not at character counts.
LLM has a strict token limit
→ Use Token-Based (Strategy 3). Counts actual tokens, not characters.
Mixed document types in pipeline
→ Use Hybrid Auto Pipeline (Strategy 9 / 10). Detects type and routes to the right chunker.
① Fixed-Size Character Chunking BASIC
Splits text at an exact character count. Fast but naive — does not respect sentence or word boundaries.
When to use: Quick experiments, when document structure doesn't matter, or as a baseline to compare other strategies against. Not recommended for production.
Visual — how fixed-size chunking looks:
Chunk 1 (chars 0–500) overlap Chunk 2 (chars 400–900) overlap Chunk 3 (chars 800–1300) overlap Chunk 4 (chars 1200–1700)
⚠️ May cut a sentence in half — e.g., "The AI mode" | "l performed well"
1
Choose chunk_size

500 chars ≈ 80 words. 1000 chars ≈ 180 words. Start with 500 and adjust.

2
Choose chunk_overlap

Set to ~15% of chunk_size. Prevents losing context at boundaries.

3
Call split_documents()

Pass your Stage 1 docs list. Each chunk = one Document with inherited metadata.

01_fixed_size_chunking.py
from langchain_text_splitters import CharacterTextSplitter

# ── Basic Fixed-Size Chunking ──────────────────────────
splitter = CharacterTextSplitter(
    separator="\n\n",    # try to split at paragraph breaks first
    chunk_size=500,      # max chars per chunk
    chunk_overlap=80,    # chars shared with next chunk
    length_function=len, # measure by character count
    is_separator_regex=False
)

# stage1_docs = list[Document] from your Stage 1 ingestion
chunks = splitter.split_documents(stage1_docs)

print(f"Original docs : {len(stage1_docs)}")
print(f"Total chunks  : {len(chunks)}")
print(f"Avg chunk size: {sum(len(c.page_content) for c in chunks)//len(chunks)} chars")

# Inspect one chunk
print(f"\n--- Chunk 0 ---")
print(f"Content: {chunks[0].page_content[:200]}")
print(f"Metadata: {chunks[0].metadata}")

# ── Varying separators ────────────────────────────────
# For CSV text, split on newline not double newline
csv_splitter = CharacterTextSplitter(
    separator="\n",
    chunk_size=300, chunk_overlap=0
)

# For dense text (no paragraphs), split on nothing (pure char count)
dense_splitter = CharacterTextSplitter(
    separator="",
    chunk_size=400, chunk_overlap=60
)
② Recursive Character Chunking RECOMMENDED DEFAULT
The best general-purpose chunker. Tries to split at paragraphs → sentences → words → characters in order. Respects natural text structure.
Use this for: PDF text, plain text files, reports, articles — basically any natural language document. This should be your default choice unless you have a specific reason to use something else.
How recursive splitting works — separator hierarchy:
Try: \n\n (paragraphs) → still too big? Try: \n (lines) → still too big? Try: " " (words) → still too big? Try: "" (chars)

def analyze_chunks(chunks, label="chunks"):
    sizes = [len(c.page_content) for c in chunks]
    print(f"\n{label}: {len(chunks)} chunks")
    print(f"  Min size : {min(sizes)}")
    print(f"  Max size : {max(sizes)}")

Study tip: Run analyze_chunks() after every strategy change. If max_size is much larger than chunk_size, your separators are too rare in the document. If min_size is tiny, your text has many empty sections — clean the data first.
③ Token-Based Chunking LLM-AWARE
Counts actual tokens (not characters) per chunk. Critical when your LLM has a token limit and you need precise control.
Why tokens matter: LLMs charge by token and have a context window limit in tokens — not characters. 1000 characters ≈ 200–300 tokens depending on language. Token-based chunking is the only way to guarantee you won't exceed the LLM's context window.

④ Semantic Chunking ADVANCED · BEST QUALITY
Splits where the meaning changes — uses your embedding model to detect topic shifts between sentences. The highest quality chunking strategy.
Use this for: Research papers, long reports, technical manuals, Wikipedia articles — any document where topics shift gradually within the text. The extra compute cost is worth it for these document types.
1
Split into sentences

The chunker first breaks the full text into individual sentences using regex.

2
Embed each sentence

Your local embedding model encodes every sentence as a vector.

3
Measure similarity between neighbours

Calculates cosine distance between each consecutive sentence pair.

4
Find breakpoints

Where similarity drops sharply (topic change) → insert a chunk boundary.

5
Merge into final chunks

Sentences between boundaries are joined into one coherent chunk.

04a_semantic_chunking_local.py — with your local embedding model
from langchain_experimental.text_splitter import SemanticChunker
from langchain_huggingface import HuggingFaceEmbeddings

# ── Load your local embedding model ──────────────────
embeddings = HuggingFaceEmbeddings(
    model_name="sentence-transformers/all-MiniLM-L6-v2",
    # Replace with your local model path if needed:
    # model_name="/path/to/your/local/model"
    model_kwargs={"device": "cpu"}  # or "cuda" if you have GPU
)

# ── Semantic Chunker: 3 threshold methods ─────────────

# Method A: percentile (default) — splits at the X% largest similarity drops
chunker_percentile = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="percentile",
    breakpoint_threshold_amount=95  # split when drop > 95th percentile
)

# Method B: standard_deviation — split when drop > X std deviations
chunker_std = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="standard_deviation",
    breakpoint_threshold_amount=3  # split if drop > 3 sigma
)

# Method C: interquartile — split using IQR distance
chunker_iq = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="interquartile"
)

# ── Apply and compare ─────────────────────────────────
for name, chunker in [
    ("percentile",        chunker_percentile),
    ("std_deviation",     chunker_std),
    ("interquartile",     chunker_iq)
]:
    chunks = chunker.split_documents(stage1_docs)
    print(f"{name:20s}: {len(chunks):4d} chunks  | avg {sum(len(c.page_content) for c in chunks)//len(chunks)} chars")

# ── Production use (pick one method) ─────────────────
semantic_chunks = chunker_percentile.split_documents(stage1_docs)
print(f"\nSemantic chunks: {len(semantic_chunks)}")
print(f"Sample:\n{semantic_chunks[0].page_content}")
04b_semantic_manual.py — manual semantic chunking without the experimental package
# If langchain_experimental is not available, implement manually
import numpy as np
from langchain.schema import Document
from sentence_transformers import SentenceTransformer
import re

model = SentenceTransformer("all-MiniLM-L6-v2")

def semantic_chunk_manual(text: str, threshold: float=0.4) -> list[str]:
    # Step 1: split into sentences
    sentences = re.split(r'(?<=[.!?])\s+', text.strip())
    sentences = [s for s in sentences if len(s) > 20]
    if len(sentences) < 2: return [text]

    # Step 2: embed all sentences
    embeddings = model.encode(sentences)

    # Step 3: cosine distance between consecutive sentences
    def cosine_dist(a, b):
        return 1 - np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

    distances = [cosine_dist(embeddings[i], embeddings[i+1])
                 for i in range(len(embeddings)-1)]

    # Step 4: find breakpoints where distance > threshold
    breakpoints = [i+1 for i, d in enumerate(distances) if d > threshold]

    # Step 5: group sentences into chunks
    chunks, start = [], 0
    for bp in breakpoints + [len(sentences)]:
        chunk = " ".join(sentences[start:bp])
        if chunk: chunks.append(chunk)
        start = bp
    return chunks

# Apply to all documents
semantic_docs = []
for doc in stage1_docs:
    text_chunks = semantic_chunk_manual(doc.page_content)
    for i, chunk_text in enumerate(text_chunks):
        semantic_docs.append(Document(
            page_content=chunk_text,
            metadata={**doc.metadata, "chunk_index": i}
        ))
print(f"Manual semantic chunks: {len(semantic_docs)}")
⑤ Paragraph-Based Chunking STRUCTURE-AWARE
Keeps entire paragraphs together. Ideal when each paragraph contains one complete idea — essays, reports, narrative documents.
Use this for: Business reports, legal documents, news articles, blog posts — any writing where paragraphs are meaningful units. Avoid for dense academic text where paragraphs are very long.
05_paragraph_chunking.py
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain.schema import Document

# ── Method A: Double newline as paragraph separator ────
# \n\n = paragraph boundary in most text files
para_splitter = RecursiveCharacterTextSplitter(
    separators=["\n\n"],   # ONLY split on double newlines
    chunk_size=2000,       # large enough to hold most paragraphs
    chunk_overlap=0        # paragraphs are self-contained; no overlap
)

chunks = para_splitter.split_documents(stage1_docs)
print(f"Paragraph chunks: {len(chunks)}")

# ── Method B: Manual paragraph extraction ─────────────
def split_into_paragraphs(docs: list[Document],
                            min_para_length: int=100,
                            max_para_length: int=2000) -> list[Document]:
    """
    Split documents into paragraphs.
    Skips paragraphs shorter than min_para_length (headers, blank lines).
    If a paragraph is longer than max_para_length, splits it at sentences.
    """
    result = []
    for doc in docs:
        paragraphs = doc.page_content.split("\n\n")
        for i, para in enumerate(paragraphs):
            para = para.strip()
            if len(para) < min_para_length:
                continue          # skip empty / header-only paragraphs
            if len(para) > max_para_length:
                # split oversized paragraphs at sentence boundaries
                sub_splitter = RecursiveCharacterTextSplitter(
                    chunk_size=max_para_length, chunk_overlap=100,
                    separators=[". ", "! ", "? ", " "]
                )
                sub_chunks = sub_splitter.create_documents(
                    [para], metadatas=[doc.metadata]
                )
                result.extend(sub_chunks)
            else:
                result.append(Document(
                    page_content=para,
                    metadata={**doc.metadata, "para_index": i}
                ))
    return result

para_docs = split_into_paragraphs(
    stage1_docs,
    min_para_length=80,
    max_para_length=1500
)
print(f"Paragraph docs (manual): {len(para_docs)}")
print(f"Sample: {para_docs[0].page_content[:300]}")
⑥ Markdown Header Chunking STRUCTURE-PRESERVING
Splits markdown documents at heading boundaries (# / ## / ###) and stores the full header hierarchy in metadata. Perfect for documentation sites, wikis, and structured guides.
Use this for: Documentation (.md), wikis, README files, technical guides — anything where headings are meaningful navigation points. The header hierarchy is stored in metadata, enabling header-filtered retrieval.
06_markdown_chunking.py
from langchain_text_splitters import (
    MarkdownHeaderTextSplitter,
    RecursiveCharacterTextSplitter
)

# ── Step 1: Split by headers ──────────────────────────
headers_to_split_on = [
    ("#",   "Header1"),   # H1 → stored in metadata as Header1
    ("##",  "Header2"),
    ("###", "Header3"),
]

md_splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=headers_to_split_on,
    strip_headers=False  # keep the header text inside the chunk content
)

# Apply to markdown docs (filter from stage1 by type)
md_docs = [d for d in stage1_docs if d.metadata.get("data_type") == "markdown"]

header_chunks = []
for doc in md_docs:
    splits = md_splitter.split_text(doc.page_content)
    for s in splits:
        s.metadata.update(doc.metadata)   # merge source metadata
    header_chunks.extend(splits)

print(f"Header-split chunks: {len(header_chunks)}")

# Each chunk now has rich metadata like:
# {'Header1': 'Introduction', 'Header2': 'Setup', 'source': 'guide.md'}
# This enables filtered retrieval: "find chunks under Header2=Setup"

# ── Step 2: Sub-split large sections ─────────────────
# Some sections may still be very long — sub-split them
sub_splitter = RecursiveCharacterTextSplitter(
    chunk_size=800, chunk_overlap=100
)

final_md_chunks = sub_splitter.split_documents(header_chunks)
print(f"Final markdown chunks (after sub-split): {len(final_md_chunks)}")

# Inspect metadata preserved through the chain
for chunk in final_md_chunks[:3]:
    print(f"  Metadata: {chunk.metadata}")
    print(f"  Content : {chunk.page_content[:100]}")
    print()
⑦ Code & HTML-Aware Chunking SYNTAX-AWARE
Language-specific splitters that understand code syntax — never splits in the middle of a function, class, or HTML tag. Supports Python, JS, HTML, CSS, and more.
Use this for: Code documentation, GitHub READMEs with code blocks, HTML pages, API documentation with code samples. Preserves code structure so the LLM can reason about complete functions.

⑧ Sentence-Window Chunking RETRIEVAL-OPTIMIZED
Split at individual sentences, but retrieve with a surrounding context window. Gives pinpoint precision at index time, richer context at retrieval time.
Use this for: Q&A systems, factoid retrieval, customer support bots — anywhere you need to find very specific facts. The sentence is indexed precisely, but the LLM gets surrounding context during generation.
1
Split into sentences

Use NLTK or spaCy to get linguistically correct sentence boundaries.

2
Embed each sentence

Index every sentence individually — this is what gets compared during retrieval.

3
Store window in metadata

Each sentence Document stores its surrounding ±N sentences in metadata.

4
Retrieve sentence, return window

The sentence matches the query, but the LLM sees the full window as context.

# ── Sentence-Window pattern ───────────────────────────
import nltk
from nltk.tokenize import sent_tokenize

def sentence_window_chunks(
    docs: list[Document],
    window_size: int = 2  # sentences on each side
) -> list[Document]:
    """
    Each chunk = 1 sentence (for indexing).
    metadata['window'] = sentence + window_size sentences on each side (for LLM context).
    """
    result = []
    for doc in docs:
        sentences = sent_tokenize(doc.page_content)
        for i, sent in enumerate(sentences):
            if len(sent.strip()) < 20:
                continue
            # build context window
            start = max(0, i - window_size)
            end   = min(len(sentences), i + window_size + 1)
            window = " ".join(sentences[start:end])

            result.append(Document(
                page_content=sent,          # ← indexed (precise)
                metadata={
                    **doc.metadata,
                    "sentence_index": i,
                    "window_context": window  # ← sent to LLM (rich)
                }
            ))
    return result

window_docs = sentence_window_chunks(stage1_docs, window_size=2)

print(f"\nSentence (indexed): {window_docs[5].page_content}")
print(f"Window  (for LLM) : {window_docs[5].metadata['window_context']}")

⑨ Hybrid Chunking Pipeline PRODUCTION PATTERN
Chain multiple chunking strategies — first split by structure (headers/paragraphs), then by size. The gold standard for production RAG systems.
When to use: Any real-world pipeline where you have mixed document types — some PDFs, some markdown docs, some HTML pages, some CSVs. This pattern routes each document to the right chunker based on its data_type metadata from Stage 1.
09_hybrid_chunking.py
from langchain_text_splitters import (
    RecursiveCharacterTextSplitter,
    MarkdownHeaderTextSplitter,
    NLTKTextSplitter,
    Language
)
from langchain.schema import Document

# ── Pattern: Structure-first, then size ───────────────
def hybrid_chunk_markdown(docs: list[Document]) -> list[Document]:
    """Split markdown by headers, then sub-split large sections."""
    md_header_splitter = MarkdownHeaderTextSplitter(
        headers_to_split_on=[("#","H1"),("##","H2"),("###","H3")]
    )
    sub_splitter = RecursiveCharacterTextSplitter(
        chunk_size=800, chunk_overlap=100
    )
    all_chunks = []
    for doc in docs:
        header_splits = md_header_splitter.split_text(doc.page_content)
        for s in header_splits: s.metadata.update(doc.metadata)
        sub_chunks = sub_splitter.split_documents(header_splits)
        all_chunks.extend(sub_chunks)
    return all_chunks

def hybrid_chunk_pdf(docs: list[Document]) -> list[Document]:
    """Recursive char split — best for PDF text."""
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=1000, chunk_overlap=150,
        separators=["\n\n", "\n", ". ", " "]
    )
    return splitter.split_documents(docs)

def hybrid_chunk_image(docs: list[Document]) -> list[Document]:
    """Sentence splitting for OCR text — noisy, sentence-level is safer."""
    splitter = NLTKTextSplitter(chunk_size=400)
    return splitter.split_documents(docs)

def hybrid_chunk_code(docs: list[Document], lang=Language.PYTHON) -> list[Document]:
    """Language-aware split — never cuts mid-function."""
    splitter = RecursiveCharacterTextSplitter.from_language(
        language=lang, chunk_size=1500, chunk_overlap=100
    )
    return splitter.split_documents(docs)

def hybrid_chunk_csv(docs: list[Document]) -> list[Document]:
    """CSV rows from Stage 1 are already single-row chunks — pass through."""
    return docs  # no further chunking needed

# ── Dispatcher: route each doc to the right chunker ──
def hybrid_chunk_dispatcher(docs: list[Document]) -> list[Document]:
    """Route documents to the correct chunker based on data_type metadata."""
    buckets = {"pdf":[], "markdown":[], "image":[], "csv":[], "excel":[], "txt":[], "html":[], "other":[]}
    for doc in docs:
        dt = doc.metadata.get("data_type", "other")
        buckets.get(dt, buckets["other"]).append(doc)

    all_chunks = []
    all_chunks.extend(hybrid_chunk_pdf(buckets["pdf"] + buckets["txt"]))
    all_chunks.extend(hybrid_chunk_markdown(buckets["markdown"]))
    all_chunks.extend(hybrid_chunk_image(buckets["image"]))
    all_chunks.extend(hybrid_chunk_csv(buckets["csv"] + buckets["excel"]))
    all_chunks.extend(hybrid_chunk_pdf(buckets["html"]))  # HTML text uses same splitter
    all_chunks.extend(hybrid_chunk_pdf(buckets["other"]))

    return all_chunks

stage2_chunking_pipeline.py — COMPLETE, production-ready
"""
RAG Pipeline — Stage 2: Chunking
Input : list[Document] from Stage 1 ingestion
Output: list[Document] (chunks) ready for Stage 3 embedding

Strategies applied per data_type:
  pdf / txt / html → RecursiveCharacterTextSplitter
  markdown         → MarkdownHeaderTextSplitter + RecursiveCharacterTextSplitter
  image (OCR)      → NLTKTextSplitter (sentence-level)
  csv / excel      → pass-through (rows already chunked)
  scanned_pdf      → NLTKTextSplitter
"""

import nltk
from collections import Counter
from langchain.schema import Document
from langchain_text_splitters import (
    RecursiveCharacterTextSplitter,
    MarkdownHeaderTextSplitter,
    NLTKTextSplitter,
)

# Download NLTK punkt (first run only)
try:
    nltk.data.find("tokenizers/punkt")
except LookupError:
    nltk.download("punkt")


# ─────────────────────────────────────────────────────
# CHUNKER DEFINITIONS
# ─────────────────────────────────────────────────────

PDF_SPLITTER = RecursiveCharacterTextSplitter(
    chunk_size=1000, chunk_overlap=150,
    separators=["\n\n", "\n", ". ", " ", ""],
    add_start_index=True
)

MD_HEADER_SPLITTER = MarkdownHeaderTextSplitter(
    headers_to_split_on=[("#","H1"),("##","H2"),("###","H3")],
    strip_headers=False
)
MD_SUB_SPLITTER = RecursiveCharacterTextSplitter(
    chunk_size=800, chunk_overlap=100
)

IMG_SPLITTER = NLTKTextSplitter(chunk_size=400)

HTML_SPLITTER = RecursiveCharacterTextSplitter(
    chunk_size=800, chunk_overlap=100,
    separators=["
", "\n\n", "\n", " "]
)


# ─────────────────────────────────────────────────────
# PER-TYPE CHUNK FUNCTIONS
# ─────────────────────────────────────────────────────

def chunk_pdfs(docs): return PDF_SPLITTER.split_documents(docs)

def chunk_markdown(docs):
    result = []
    for doc in docs:
        header_splits = MD_HEADER_SPLITTER.split_text(doc.page_content)
        for s in header_splits: s.metadata.update(doc.metadata)
        result.extend(MD_SUB_SPLITTER.split_documents(header_splits))
    return result

def chunk_images(docs): return IMG_SPLITTER.split_documents(docs)

def chunk_html(docs):   return HTML_SPLITTER.split_documents(docs)

def chunk_passthrough(docs): return docs  # CSV/Excel: rows are chunks


# ─────────────────────────────────────────────────────
# MAIN PIPELINE
# ─────────────────────────────────────────────────────

def run_chunking(stage1_docs: list[Document]) -> list[Document]:
    """
    Main Stage 2 entry point.
    Takes Stage 1 documents, applies the right chunker per type,
    returns unified list of chunks for Stage 3 (embedding).
    """
    print(f"\n{'='*52}")
    print(f" RAG Stage 2 — Chunking ({len(stage1_docs)} input documents)")
    print(f"{'='*52}")

    # Route documents by data_type
    buckets = {"pdf":[], "txt":[], "markdown":[], "image":[],
               "scanned_pdf":[], "html":[], "csv":[], "excel":[], "json":[], "other":[]}

    for doc in stage1_docs:
        dtype = doc.metadata.get("data_type", "other")
        buckets.get(dtype, buckets["other"]).append(doc)

    all_chunks = []

    # Apply chunkers per type
    routing = [
        ("pdf",         chunk_pdfs,         buckets["pdf"]       ),
        ("txt",         chunk_pdfs,         buckets["txt"]       ),
        ("markdown",    chunk_markdown,    buckets["markdown"]  ),
        ("image",       chunk_images,      buckets["image"]     ),
        ("scanned_pdf", chunk_images,      buckets["scanned_pdf"]),
        ("html",        chunk_html,        buckets["html"]      ),
        ("csv",         chunk_passthrough, buckets["csv"]       ),
        ("excel",       chunk_passthrough, buckets["excel"]     ),
        ("json",        chunk_passthrough, buckets["json"]      ),
        ("other",       chunk_pdfs,        buckets["other"]     ),
    ]

    for dtype, chunker_fn, docs in routing:
        if not docs: continue
        try:
            chunks = chunker_fn(docs)
            all_chunks.extend(chunks)
            print(f"  [{dtype:12s}] {len(docs):3d} docs  → {len(chunks):4d} chunks")
        except Exception as e:
            print(f"  [{dtype:12s}] ERROR: {e}")

    # Post-processing: remove empty chunks
    all_chunks = [c for c in all_chunks if c.page_content.strip()]

    print(f"{'='*52}")
    print(f"✅ Total chunks ready for embedding: {len(all_chunks)}")
    sizes = [len(c.page_content) for c in all_chunks]
    print(f"   Min chars: {min(sizes)} | Max: {max(sizes)} | Avg: {sum(sizes)//len(sizes)}")
    print(f"{'='*52}\n")

    return all_chunks   # ← passes to Stage 3 (embedding)
